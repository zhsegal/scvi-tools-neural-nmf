# Five-modality factor-model benchmark — across populations & datasets

Each subfolder runs the **same** 6-model × 5-modality pipeline as
`../benchmark_modalities_haniffa_cd8.ipynb` on a different clean immune population.
A notebook differs from the CD8 reference **only in its Cell-1 config header**
(data path, gene-set library, batch/metadata columns, cache dirs); cells 2–10 are
identical. Models train fresh into `.model_cache_<run>/` on first run, then load.

## Metadata-rich studies (strong mod5 design factors)

Two extra studies, **2 cell types each**, chosen for a strong experimental-design factor
(beyond `cell_type`) so the factor↔metadata modality (mod5) has real biology to track:

| Run | Study | Cell type | Design factor (BIO_VARS) | Source |
|---|---|---|---|---|
| `perez_sle_mono` / `perez_sle_cd4` | Perez 2022 SLE lupus | classical monocyte / CD4 T | `disease` (SLE vs normal), `self_reported_ethnicity` | CELLxGENE Census `218acb0f-…` |
| `combat_mono` / `combat_cd8` | COMBAT blood atlas (Cell 2022) | classical monocyte (cMono) / CD8 T | `Source` = healthy/COVID mild·severe·critical/sepsis/influenza | Zenodo `record/6120249` |

Build the h5ads (once, on the GPU node): `python ../_download_immune_datasets.py perez_sle_mono_clean`
(and `perez_sle_cd4_clean`); `python ../_download_combat.py`. Then regenerate notebooks
(`python _gen_notebooks.py`) and run as below.

## Run order (on the GPU kernel, `neural_nmf_env`)

1. Build the Haniffa subsets (once): `python ../_make_haniffa_subsets.py`
   → writes `../haniffa_{cd4,mono,nk,b}_clean.h5ad`.
2. Build the NK gene set (once): `python ../_build_lib2_nk.py` → `../lib2_nk.gmt`. *(already built)*
3. Run a Haniffa notebook **first** so the Geneformer (~415 MB) and GenePT (~574 MB)
   embedding bundles download once and cache to `../*.pt`; then the rest:
   `bash ../_run_nb.sh modalities/<run>/benchmark_modalities_<run>.ipynb`

Semantic maps already on disk: `monocytes_clean_*`, `zheng_cd8_clean_*`.
Built on first notebook run: `tonsil_b_clean_*`, `haniffa_{cd4,mono,nk,b}_clean_*`.

## Notes

- `cellxgene_mono` has a single `cell_type`/`disease` → the *biological* side of the
  metadata modality is degenerate there; the other four modalities carry the comparison.
- Haniffa subsets use `BATCH_KEY="Site"`; CELLxGENE datasets use `donor_id`.
- Notebooks are generated by `_gen_notebooks.py` from the CD8 template — edit the template
  or that script and regenerate rather than hand-editing all seven.
- Each run writes `../benchmark_results/<run>_modalities/report_modalities.html`.
